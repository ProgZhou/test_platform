# 产品需求文档

## 1. 需求背景

在软件研发过程中，测试团队面临以下痛点：

- 测试用例分散管理，缺乏统一的录入、分组和检索平台，导致用例复用率低、维护成本高。
- 测试接口的定义和调试依赖外部工具（如 Postman），缺少与用例、脚本的联动能力。
- 测试脚本的管理和执行依赖本地环境，无法在线协作编辑和一键执行。
- 测试结果缺乏统一的报告生成和下载能力，沟通成本高。

本平台旨在提供一站式的自动化测试管理能力，覆盖用例管理、接口管理、脚本管理、自动化执行和报告生成的完整链路，降低测试团队的工具切换成本，提升测试效率。

## 2. 用户故事

- 作为 **测试工程师**，我想要 **在平台上手动创建或通过 Excel 批量导入测试用例**，以便 **快速完成用例录入，减少重复劳动**。
- 作为 **测试工程师**，我想要 **通过多级目录对测试用例进行分组管理**，以便 **按项目、小组、需求等维度组织用例，方便检索和维护**。
- 作为 **测试工程师**，我想要 **在平台上录入和管理测试接口（支持 OpenAPI 定义和表单录入）**，以便 **统一管理接口信息并直接在平台上调试**。
- 作为 **测试工程师**，我想要 **上传 Go 或 Python 测试脚本并在线调试、修改**，以便 **无需切换本地环境即可完成脚本开发和验证**。
- 作为 **测试工程师**，我想要 **一键执行自动化测试并生成测试报告**，以便 **快速获取测试结果并分享给团队**。
- 作为 **测试负责人**，我想要 **下载测试报告**，以便 **归档或在团队内流转评审**。

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 测试用例管理

- [ ] 功能点 1: **手动创建测试用例** — 用户可通过页面表单录入测试用例，包括用例名称、前置条件、操作步骤、预期结果等字段。
- [ ] 功能点 2: **Excel 批量导入测试用例** — 支持通过指定格式的 Excel 文件批量导入测试用例，平台需提供模板下载。
- [ ] 功能点 3: **多级目录分组** — 支持无限层级的树形目录结构，用于组织测试用例。目录层级参考：
  ```
  root
  └── {项目}
      └── {小组}
          └── {需求}
              └── 测试用例
  ```
  用户可自由创建、重命名、删除目录节点。
- [ ] 功能点 4: **用例编辑** — 已录入的测试用例支持修改所有字段。
- [ ] 功能点 5: **用例删除** — 支持删除单条或批量删除测试用例，删除前需二次确认。
- [ ] 功能点 6: **用例列表查看** — 支持按目录浏览、搜索和筛选已有测试用例。

#### 3.1.2 测试接口管理

- [ ] 功能点 7: **OpenAPI 格式导入接口** — 支持通过粘贴或上传 JSON/YAML 格式的 OpenAPI 3.x 标准定义来录入接口信息，平台自动解析出接口地址、请求方法、参数等。
- [ ] 功能点 8: **表单录入接口** — 支持通过页面表单逐项录入接口信息，包括：
  - 接口地址（URL）
  - 请求方法（GET/POST/PUT/DELETE 等）
  - 参数列表，每个参数包含：
    | 字段 | 说明 | 校验规则 |
    |------|------|----------|
    | 参数名称 | 参数的标识名 | 只能由大小写英文字母、下划线、0-9 数字组成，即匹配正则 `^[a-zA-Z_][a-zA-Z0-9_]*$` |
    | 参数类型 | 参数的数据类型 | 取值范围：`string`, `bool`, `int`, `long`, `float`, `object`, `array<string>`, `array<int>`, `array<long>`, `array<float>`, `array<object>` |
    | 参数说明 | 参数的文字描述 | 无特殊限制 |
    | 是否必填 | 该参数是否必传 | 布尔值：true / false |
    | 参数位置 | 参数所在位置 | 取值范围：`body`（请求体）、`header`（请求头） |
- [ ] 功能点 9: **接口在线调试** — 录入的接口支持在页面上填写参数值后发起真实请求，查看响应结果（状态码、响应体、响应头、耗时）。
- [ ] 功能点 10: **接口编辑** — 已录入的接口信息支持修改。
- [ ] 功能点 11: **接口删除** — 支持删除接口，删除前需二次确认。
- [ ] 功能点 12: **接口列表查看** — 支持浏览和搜索已录入的接口。

#### 3.1.3 测试脚本管理

- [ ] 功能点 13: **脚本上传** — 支持用户上传测试脚本文件，当前支持的语言：Go（`.go`）、Python（`.py`）。脚本的组织方式与测试用例相同，采用多级目录结构。
- [ ] 功能点 14: **脚本在线查看与修改** — 上传后的脚本支持在线代码编辑器中查看和修改（需支持语法高亮）。
- [ ] 功能点 15: **脚本在线调试** — 支持在平台上直接运行脚本并查看控制台输出和执行结果。
- [ ] 功能点 16: **脚本列表查看** — 支持按目录浏览已上传的测试脚本。

#### 3.1.4 自动化执行

- [ ] 功能点 17: **一键执行** — 用户选择测试用例和关联的测试脚本后，点击执行，平台根据测试用例自动调用脚本执行测试，并记录执行过程和结果。
- [ ] 功能点 18: **执行前校验** — 执行前需校验用户是否已上传测试用例和测试脚本，若缺失则给出明确提示。

#### 3.1.5 测试报告

- [ ] 功能点 19: **测试报告生成** — 自动化执行完成后，平台自动生成测试报告，包含执行概要（通过/失败/总数）、每条用例的执行详情和耗时。
- [ ] 功能点 20: **测试报告下载** — 生成的测试报告支持用户下载（建议支持 PDF 或 Excel 格式）。

### 3.2 边缘场景

- 场景 1: **Excel 导入格式错误** — 用户上传的 Excel 文件格式不符合模板要求（如缺少必填列、数据类型不匹配），平台应给出逐行错误提示，支持部分成功导入。
- 场景 2: **OpenAPI 定义解析失败** — 用户粘贴的 JSON/YAML 内容不符合 OpenAPI 3.x 规范，平台应提示具体的解析错误位置和原因。
- 场景 3: **参数名称校验不通过** — 用户在表单录入接口参数时，参数名称包含中文、特殊字符等非法字符，平台应实时提示校验错误。
- 场景 4: **删除非空目录** — 用户尝试删除包含子目录或用例/脚本的目录时，平台应提示该目录非空，要求用户确认是否级联删除。
- 场景 5: **上传不支持的脚本语言** — 用户上传非 `.go` 或 `.py` 后缀的文件，平台应拒绝并提示当前仅支持 Go 和 Python。
- 场景 6: **脚本执行超时** — 在线调试或自动化执行时，脚本运行超过预设时间上限，平台应终止执行并返回超时提示。
- 场景 7: **并发执行冲突** — 同一用例/脚本被多人同时触发执行时，平台应合理排队或隔离执行环境，避免结果混乱。
- 场景 8: **大文件上传** — 用户上传过大的脚本文件或 Excel 文件，平台应限制文件大小并给出提示（建议上限 10MB）。

## 4. 非功能需求

- **性能**:
  - 页面操作（用例/接口/脚本的增删改查）响应时间不超过 2 秒。
  - Excel 导入 1000 条用例应在 30 秒内完成。
  - 接口调试请求超时上限建议 30 秒。
  - 脚本在线调试执行超时上限建议 5 分钟。
  - 自动化批量执行超时上限根据用例数量动态计算，单条用例上限 5 分钟。

- **安全**:
  - 脚本执行需在沙箱环境中运行，禁止访问宿主机敏感资源。
  - 接口调试的请求应通过后端代理发出，避免前端直接发起跨域请求导致安全风险。
  - 上传文件需进行类型和大小校验，防止恶意文件上传。

- **可用性**:
  - 目录树支持拖拽排序和展开/折叠操作。
  - 代码编辑器需支持语法高亮和基本的代码提示。
  - 关键操作（删除、批量导入）需有二次确认弹窗。
  - Excel 导入需提供模板下载入口，降低用户学习成本。

## 5. 验收标准

### 测试用例管理

- [ ] AC-1: Given 用户在某个目录下点击"新建用例"，When 填写用例名称、前置条件、操作步骤、预期结果并提交，Then 用例成功创建并出现在当前目录的用例列表中。
- [ ] AC-2: Given 用户准备了一份符合模板格式的 Excel 文件，When 在目标目录下点击"导入"并上传该文件，Then 所有用例成功导入并显示在列表中。
- [ ] AC-3: Given 用户上传了一份格式错误的 Excel 文件（如缺少"用例名称"列），When 执行导入，Then 平台提示具体的错误信息，合法行正常导入，非法行跳过并列出错误详情。
- [ ] AC-4: Given 已存在一条测试用例，When 用户点击编辑并修改用例名称后保存，Then 用例名称更新成功，列表中显示新名称。
- [ ] AC-5: Given 已存在一条测试用例，When 用户点击删除并在确认弹窗中点击"确定"，Then 用例从列表中移除。
- [ ] AC-6: Given 用户在根目录下，When 依次创建"项目A > 小组B > 需求C"三级目录，Then 目录树正确展示三级嵌套结构，且可在"需求C"下创建用例。

### 测试接口管理

- [ ] AC-7: Given 用户粘贴了一段合法的 OpenAPI 3.x YAML 定义，When 点击"导入"，Then 平台解析出所有接口并展示在接口列表中，包括接口地址、请求方法和参数信息。
- [ ] AC-8: Given 用户粘贴了一段非法的 YAML 内容，When 点击"导入"，Then 平台提示解析失败及具体错误原因。
- [ ] AC-9: Given 用户通过表单录入接口参数，When 参数名称输入"用户名"（含中文），Then 表单实时提示"参数名称只能包含英文字母、数字和下划线"。
- [ ] AC-10: Given 用户通过表单录入接口参数，When 参数类型选择"array<string>"、参数位置选择"body"，Then 参数保存成功，接口详情中正确显示该参数配置。
- [ ] AC-11: Given 已录入一个 GET 接口，When 用户在调试页面点击"发送请求"，Then 平台发起真实请求并展示响应状态码、响应体、响应头和耗时。
- [ ] AC-12: Given 已录入一个接口，When 用户点击删除并确认，Then 接口从列表中移除。

### 测试脚本管理

- [ ] AC-13: Given 用户选择了一个 `.py` 文件，When 在目标目录下点击"上传"，Then 脚本上传成功并出现在脚本列表中。
- [ ] AC-14: Given 用户选择了一个 `.js` 文件，When 点击"上传"，Then 平台拒绝上传并提示"当前仅支持 Go (.go) 和 Python (.py) 脚本"。
- [ ] AC-15: Given 已上传一个 Python 脚本，When 用户在在线编辑器中修改代码并保存，Then 脚本内容更新成功，再次打开显示修改后的内容。
- [ ] AC-16: Given 已上传一个 Python 脚本（如 `print("hello")`），When 用户点击"调试运行"，Then 平台执行脚本并在控制台区域显示输出 `hello`。

### 自动化执行与报告

- [ ] AC-17: Given 用户已上传测试用例和关联的测试脚本，When 点击"执行"，Then 平台自动逐条执行测试并在完成后生成测试报告。
- [ ] AC-18: Given 用户仅上传了测试用例但未上传测试脚本，When 点击"执行"，Then 平台提示"请先上传测试脚本"。
- [ ] AC-19: Given 测试执行完成并生成了报告，When 用户点击"下载报告"，Then 浏览器下载测试报告文件（PDF 或 Excel 格式）。
- [ ] AC-20: Given 某条测试脚本执行超过 5 分钟，When 达到超时上限，Then 平台终止该脚本执行，标记为"超时"，并继续执行后续用例。

## 6. 依赖与约束

### 依赖的外部系统
- 脚本执行运行时环境：需要后端支持 Go 和 Python 的运行时（如 Docker 容器化沙箱）。
- 文件存储服务：用于存储上传的脚本文件、Excel 文件和生成的测试报告。
- OpenAPI 解析库：用于解析 JSON/YAML 格式的 OpenAPI 3.x 定义（如 swagger-parser 或同类库）。

### 技术约束
- 脚本语言支持范围：V1 版本仅支持 Go 和 Python，后续版本可扩展。
- 文件上传大小限制：单文件不超过 10MB。
- 脚本执行必须在隔离的沙箱环境中运行，不得直接在宿主机上执行用户上传的代码。
- 目录层级理论上无限制，但建议前端展示时做合理的层级深度提示（如超过 10 级给出警告）。
- 参数名称校验规则：`^[a-zA-Z_][a-zA-Z0-9_]*$`，即必须以字母或下划线开头，后续可包含字母、数字、下划线。
